## Per-Site Configuration

Each project can override global settings with site-specific configuration:

1. Open your project
2. Navigate to **Tunnel** tab
3. Click **Configure** to set custom options
4. Save and start tunnel!

### Available Options

- **Site Authtoken** - Use a different ngrok account for this project
- **Custom Domain** - Specify a custom domain (requires paid plan)
- **Region** - Select optimal region for your audience
- **Compression** - Enable/disable compression per site

<Callout type="tip">
  With a paid ngrok plan, use custom domains like `api.yourdomain.com` for demos.
</Callout>

## Using Tunnels

### Starting a Tunnel

1. Ensure your project devcontainer is running (open in VS Code)
2. Navigate to the project in DAMP
3. Click the **Tunnel** tab
4. Click **Start Tunnel**
5. Copy the public URL and share!

## Advanced Features

### Custom Regions

Optimize latency by selecting the closest region:

- **us** - United States (default)
- **eu** - Europe
- **ap** - Asia/Pacific
- **au** - Australia
- **sa** - South America
- **jp** - Japan
- **in** - India

### Compression

Enable compression to reduce bandwidth:

```json
{
  "compression": true
}
```

Automatically compresses HTTP responses using gzip.

### Multiple Tunnels

Run tunnels for multiple projects simultaneously:

```
Project A: https://app-a.ngrok.io
Project B: https://app-b.ngrok.io
Project C: https://app-c.ngrok.io
```

Each project gets its own isolated tunnel.

## Security Considerations

<Callout type="warning">
  - Tunnels expose your local environment to the internet
  - Anyone with the URL can access your site
  - Use .env protection for sensitive data
  - Don't share tunnel URLs publicly unless intended
</Callout>

### Best Practices

Do:
- Use tunnels for temporary sharing
- Add authentication to sensitive endpoints
- Monitor tunnel traffic
- Stop tunnels when not in use

Don't:
- Share tunnel URLs in public forums
- Use production credentials in dev
- Leave tunnels running indefinitely
- Expose admin panels without protection

## Troubleshooting

<Accordions type="single">
  <Accordion title="Tunnel Won't Start">
    <b>Problem:</b> "Devcontainer is not running"<br/><br/>
    <b>Solution:</b> Open your project in VS Code and reopen in container first.<br/><br/>
    <b>Problem:</b> "Authtoken not configured"<br/><br/>
    <b>Solution:</b> Add your ngrok authtoken in Settings â†’ Ngrok Tunneling.<br/><br/>
    <b>Problem:</b> "Tunnel already running"<br/><br/>
    <b>Solution:</b> Stop the existing tunnel before starting a new one.
  </Accordion>
  <Accordion title="Can't Access Tunnel URL">
    <b>Problem:</b> Tunnel URL returns 502 Bad Gateway<br/><br/>
    <b>Solutions:</b>
    <ol>
      <li>Verify devcontainer is running and serving on port 8080</li>
      <li>Check that your application is listening on <code>0.0.0.0</code> (not <code>localhost</code>)</li>
      <li>Restart the tunnel</li>
    </ol>
  </Accordion>
  <Accordion title="Slow Connection">
    <b>Problem:</b> Tunnel feels slow<br/><br/>
    <b>Solutions:</b>
    <ol>
      <li>Choose a closer region in tunnel configuration</li>
      <li>Enable compression</li>
      <li>Check ngrok status page for service issues</li>
    </ol>
  </Accordion>
</Accordions>
