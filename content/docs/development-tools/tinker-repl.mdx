## What is Tinker?

Tinker is Laravel's REPL (Read-Eval-Print Loop) powered by PsySH. It provides an interactive console where you can:

- Execute PHP code instantly
- Interact with your Laravel application
- Query the database using Eloquent
- Test methods and functions
- Debug and troubleshoot

<Callout type="info">
  Tinker runs inside your devcontainer with full access to your Laravel application, including database connections, environment variables, and all your code.
</Callout>

1. Checks that your devcontainer is running
2. Opens a new terminal window
3. Executes: `docker exec -it {project}_devcontainer php artisan tinker`
4. You're in Tinker - ready to code

## Tips & Tricks

### Show Available Commands

```php
>>> help
```

Lists all available Tinker commands and shortcuts.

### Clear Screen

```php
>>> clear
```

Or press `Ctrl+L` to clear the terminal.

### View Variables

```php
>>> show $user
```

Displays formatted output of any variable.

### Multi-Line Code

Press `Enter` without completing statement:

```php
>>> $users = User::where('active', true)
...           ->orderBy('created_at', 'desc')
...           ->take(10)
...           ->get();
```

### History

Use â†‘ and â†“ arrow keys to navigate through command history.

### Exit Tinker

```php
>>> exit
```

Or press `Ctrl+D` to quit Tinker.

<Callout type="tip">
  Tinker session state is not persistent. Save complex queries in a file for later use.
</Callout>

## Requirements

### Devcontainer Must Be Running

Before clicking "Open Tinker":

1. Open your project in VS Code
2. Run "Dev Containers: Reopen in Container"
3. Wait for container to fully start
4. Now click "Open Tinker" in DAMP

## Troubleshooting

<Accordions>

<Accordion title="Tinker Button Not Available">

**Problem**: Can't find "Open Tinker" button

**Solutions:**
- Ensure project is a Laravel project
- Check that site type is set to "Laravel"
- Verify Laravel installation in project directory

</Accordion>

<Accordion title="Devcontainer Not Running Error">

**Problem**: "Devcontainer is not running"

**Solutions:**
1. Open project in VS Code
2. Press `Ctrl+Shift+P`
3. Run "Dev Containers: Reopen in Container"
4. Wait for container to start fully
5. Try opening Tinker again

</Accordion>

<Accordion title="Tinker Not Available">

**Problem**: "Tinker is not available"

**Solutions:**
- Install Tinker: `composer require laravel/tinker`
- Rebuild devcontainer after installation
- Verify Laravel version compatibility

</Accordion>
