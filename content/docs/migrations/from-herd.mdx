## Migration Steps

<Steps>

<Step>
**Stop Herd Services**

Free ports 80 and 443:

1. Open Laravel Herd
2. Click "Stop All Services"
3. Quit Herd completely

</Step>

<Step>
**Add Project to DAMP**

1. Launch DAMP
2. Click "Add Project"
3. Select "Existing Project"
4. Configure:
   - **Name**: Your project name
   - **Path**: Your project folder (e.g., `C:\Herd\my-app`)
   - **PHP Version**: Match your current version

</Step>

<Step>
**Install Services**

Go to Services page and install required services:

- **MySQL** - For database
- **Redis** - For caching/queues
- **Mailpit** - For email testing

</Step>

<Step>
**Migrate Database (Optional)**

If you have existing data, export and import:

**Option 1: Command Line**

```bash
# Export from Herd MySQL
mysqldump -u root -p your_database > backup.sql

# Import to DAMP (inside VS Code DevContainer)
mysql -h damp-mysql -u damp -pdamp damp < backup.sql
```

**Option 2: GUI Tools**

Use database management tools like TablePlus, DBeaver, or HeidiSQL:

1. **Export from Herd**: Connect to `localhost:3306`, export database as SQL
2. **Import to DAMP**: Get MySQL port from DAMP Services page, connect to `localhost:{port}`, import SQL file

Connection details for DAMP MySQL:

- **Host**: `localhost`
- **Port**: Check DAMP app (Services page)
- **Username**: `damp`
- **Password**: `damp`
- **Database**: `damp`

</Step>

<Step>
**Update Configuration**

Open project in VS Code DevContainer and update `.env`:

```
# Update database connection
DB_HOST=damp-mysql
DB_DATABASE=damp
DB_USERNAME=damp
DB_PASSWORD=damp

# Update Redis (if using)
REDIS_HOST=damp-redis

# Update Mail (if using)
MAIL_HOST=damp-mailpit
MAIL_PORT=1025
```

</Step>

<Step>
**Test Project**

1. Click "Open in VS Code" in DAMP
2. Reopen in Container when prompted
3. Browse to `https://yourproject.local`
4. Install SSL certificate if prompted

</Step>

</Steps>

## Configuration Changes

### Generated Files

DAMP adds these files to your project:

```
your-project/
├── .devcontainer/
│   └── devcontainer.json
├── .vscode/
│   └── launch.json
├── Dockerfile
├── docker-compose.yml
└── .dockerignore
```

### Service Hostnames

Service names are automatically generated by Docker. Get actual service names and ports from the DAMP app (Services page).

Default pattern:

| Service  | Herd        | DAMP           |
| -------- | ----------- | -------------- |
| Database | `127.0.0.1` | `damp-mysql`   |
| Redis    | `127.0.0.1` | `damp-redis`   |
| Mail     | `127.0.0.1` | `damp-mailpit` |

### Domain Format

| Tool | Format                  |
| ---- | ----------------------- |
| Herd | `http://project.test`   |
| DAMP | `https://project.local` |

### Commands

Run commands directly in DevContainer (no wrapper needed):

```bash
php artisan migrate
composer install
npm run dev
```

## Troubleshooting

<Accordions>
  <Accordion title="Port Conflict">
    Caddy can't start: 1. Right-click Herd tray icon → Quit Herd 2. Restart DAMP
  </Accordion>
  <Accordion title="Database Connection Failed">
    Can't connect to MySQL: 1. Verify MySQL service is installed in DAMP 2.
    Check `.env` has `DB_HOST=damp-mysql` 3. Restart DevContainer
  </Accordion>
  <Accordion title="Project Not Loading">
    Site doesn't load at `https://project.local`: 1. Verify project status is
    "Running" in DAMP 2. Check DevContainer is active in VS Code 3. Install SSL
    certificate from DAMP
  </Accordion>
</Accordions>
