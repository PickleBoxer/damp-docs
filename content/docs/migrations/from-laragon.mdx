## Migration Steps

<Steps>

<Step>
**Stop Laragon Services**

Free ports 80 and 443:

1. Open Laragon
2. Click "Stop All"
3. Right-click system tray icon → Exit
4. Verify ports are free (Windows):
   ```powershell
   netstat -ano | findstr ":80 :443"
   ```

</Step>

<Step>
**Add Project to DAMP**

1. Launch DAMP
2. Click "Add Project"
3. Select "Existing Project"
4. Configure:
   - **Name**: Your project name
   - **Path**: Your project folder (e.g., `C:\laragon\www\my-project`)
   - **PHP Version**: Match your current version

</Step>

<Step>
**Install Services**

Go to Services page and install required services:

- **MySQL** - For database
- **PostgreSQL** - If using PostgreSQL
- **Redis** - For caching/queues
- **Mailpit** - For email testing

</Step>

<Step>
**Migrate Database (Optional)**

If you have existing data, export and import:

**Option 1: Command Line**

```bash
# Export from Laragon MySQL
mysqldump -u root -p your_database > backup.sql

# Import to DAMP (inside VS Code DevContainer)
mysql -h damp-mysql -u damp -pdamp damp < backup.sql
```

**Option 2: GUI Tools**

Use database management tools like TablePlus, DBeaver, or HeidiSQL:

1. **Export from Laragon**: Connect to `localhost:3306`, export database as SQL
2. **Import to DAMP**: Get MySQL port from DAMP Services page, connect to `localhost:{port}`, import SQL file

Connection details for DAMP MySQL:

- **Host**: `localhost`
- **Port**: Check DAMP app (Services page)
- **Username**: `damp`
- **Password**: `damp`
- **Database**: `damp`

</Step>

<Step>
**Update Configuration**

Open project in VS Code DevContainer and update configuration:

**Laravel .env:**

```
# Update database connection
DB_HOST=damp-mysql
DB_DATABASE=damp
DB_USERNAME=damp
DB_PASSWORD=damp

# Update Redis (if using)
REDIS_HOST=damp-redis

# Update Mail (if using)
MAIL_HOST=damp-mailpit
MAIL_PORT=1025
```

**Plain PHP:**

```php
// Update database host
$host = 'damp-mysql';
$db = 'damp';
$user = 'damp';
$pass = 'damp';
```

</Step>

<Step>
**Test Project**

1. Click "Open in VS Code" in DAMP
2. Reopen in Container when prompted
3. Browse to `https://yourproject.local`
4. Install SSL certificate if prompted

</Step>

</Steps>

## Configuration Changes

### Generated Files

DAMP adds these files to your project:

```
your-project/
├── .devcontainer/
│   └── devcontainer.json
├── .vscode/
│   └── launch.json
├── Dockerfile
├── docker-compose.yml
└── .dockerignore
```

### Service Hostnames

Service names are automatically generated by Docker. Get actual service names and ports from the DAMP app (Services page).

Default pattern:

| Service  | Laragon     | DAMP           |
| -------- | ----------- | -------------- |
| Database | `127.0.0.1` | `damp-mysql`   |
| Redis    | `127.0.0.1` | `damp-redis`   |
| Mail     | `127.0.0.1` | `damp-mailpit` |

### Domain Format

| Tool    | Format                  |
| ------- | ----------------------- |
| Laragon | `http://project.test`   |
| DAMP    | `https://project.local` |

### Commands

Run commands directly in DevContainer (no wrapper needed):

```bash
php artisan migrate
composer install
npm run dev
```

## Troubleshooting

<Accordions>
  <Accordion title="Port Conflict">
  Caddy can't start:
  1. Fully exit Laragon (not just stop)
  2. Check for Apache/IIS:
     ```powershell
     Get-Process | Where-Object {$_.ProcessName -like "*apache*" -or $_.ProcessName -like "*nginx*"}
     ```
  3. Stop conflicting services
  </Accordion>
  <Accordion title="Database Connection Failed">
  Can't connect to MySQL:
  1. Verify MySQL service is installed in DAMP
  2. Check `.env` has `DB_HOST=damp-mysql`
  3. Import database if needed
  4. Restart DevContainer
  </Accordion>
  <Accordion title="Missing Extensions">
  PHP extension not found:
  1. Edit `.devcontainer/Dockerfile`
  2. Add extension: `RUN docker-php-ext-install <extension>`
  3. Rebuild: Ctrl+Shift+P → "Rebuild Container"
  </Accordion>
  <Accordion title="Slow Performance">
  Site is slower than Laragon:
  1. Click project in DAMP
  2. Click "Sync Volume"
  3. Choose excluded folders (node_modules, vendor)

  Note: Volume sync is bidirectional - changes sync from local filesystem to named Docker volume and vice versa.
  </Accordion>
</Accordions>
