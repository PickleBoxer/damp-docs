## Project Actions

### Open in VS Code

Launch directly into VS Code:

1. Click "Open in VS Code" button
2. VS Code opens the project
3. Reopen in Container when prompted
4. Start coding

<Callout type="tip">
  This launches your project in VS Code without folder browsing.
</Callout>

### Open Folder

Open the project folder in Windows Explorer:

- View project files
- Copy/paste files
- Backup project manually

### Browse Project

Visit the project in your default browser:

```
https://myproject.local
```

<Callout type="warning">
  Project must be running in VS Code DevContainer to respond to browser requests.
</Callout>

### Environment Variables

DB_PORT=3306

## Volume Sync

Named volumes provide **fast performance** but files live in the container. Sync copies them to your local machine.

**Quick Action:**
- **Sync From Volume** - Copy container files to local folder

**Options:**
- Exclude `node_modules/` - Skip Node.js dependencies
- Exclude `vendor/` - Skip PHP dependencies

<Callout type="info">
  See the complete [Volume Sync Guide](/docs/volume-sync) for details on performance, sync timing, and troubleshooting.
</Callout>

## Deleting Projects

### Safe Deletion

When you delete a project, DAMP:

1. ‚úÖ Stops the DevContainer (if running)
2. ‚úÖ Removes project from DAMP dashboard
3. ‚úÖ Removes domain from hosts file
4. ‚úÖ Removes SSL certificate
5. ‚úÖ Removes Caddy configuration
6. ‚úÖ **Preserves project folder**
7. ‚úÖ **Preserves named volume**

<Callout type="success">
  Your code and data are safe - only DAMP's tracking and configuration are removed.
</Callout>

### Delete Process

<Steps>

<Step>
**Select Project**

Click the project you want to remove
</Step>

<Step>
**Click Delete**

Click "Delete Project" button
</Step>

<Step>
**Confirm**

Confirm deletion in dialog
</Step>

<Step>
**Done**

Project removed from DAMP - folder and volume remain on disk
</Step>

</Steps>

### What Remains After Deletion

Your local filesystem still contains:

```
C:\Users\You\Projects\myproject\    ‚Üê Project folder intact
```

Docker volumes still exist:

```bash
docker volume ls
# Shows: myproject_data
```

### Complete Removal

To fully remove everything:

```powershell
# 1. Delete from DAMP (above steps)

# 2. Delete project folder
Remove-Item -Path "C:\Users\You\Projects\myproject" -Recurse -Force

# 3. Delete Docker volume
docker volume rm myproject_data
```

## Troubleshooting

<Accordions>
  <Accordion title="Project Shows 'Created' Instead of 'Running'">
    **Problem:** Project exists but shows red "Created" status

    **Solution:**
    1. Open project in VS Code
    2. Reopen in Container
    3. Wait for DevContainer to fully start
    4. Refresh DAMP
  </Accordion>
  <Accordion title="Project Not Accessible in Browser">
    **Problem:** `https://myproject.local` doesn't load

    **Solutions:**
    - ‚úÖ Check project status is "Running" (green)
    - ‚úÖ Verify DevContainer is active in VS Code
    - ‚úÖ Trust SSL certificate in DAMP
    - ‚úÖ Check Caddy service is running
    - ‚úÖ Clear browser cache
  </Accordion>
  <Accordion title="Volume Sync Not Working">
    **Problem:** Sync fails or files don't update

    **Solutions:**
    - ‚úÖ Close project in VS Code before syncing
    - ‚úÖ Check Docker Desktop is running
    - ‚úÖ Verify volume exists: `docker volume ls`
    - ‚úÖ Try opposite sync direction
  </Accordion>
</Accordions>
